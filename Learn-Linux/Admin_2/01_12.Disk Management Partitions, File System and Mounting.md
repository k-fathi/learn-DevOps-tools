# Disk Management: Partitions, File System, and Mounting

---

## Partition Tables

### MBR: Master Boot Record

- **Commands:**
    - `lsblk` — List block devices.
    - `cat /proc/partitions` — Show partition info.
    - `fdisk /dev/&lt;DiskName&gt;` — Manage MBR partitions.
    - `partprobe` — Inform OS of partition table changes.

---

### GPT: GUID Partition Table

- **Commands:**
    - `lsblk` — List block devices.
    - `parted /dev/&lt;DiskName&gt;` — Manage GPT partitions.
    - `help` — Get help in parted.
    - `udevadm settle` — Wait for device events.

---

## File System Types

| Type   | Description |
|--------|-------------|
| ext4   | Widely used, stable, supports large volumes, journaling, extents. Backward compatible with ext3/ext2. Not as scalable as XFS. |
| xfs    | High performance, scalable, reliable. Ideal for large files/storage. Supports journaling, snapshots, dynamic inode allocation. Cannot be shrunk. Common in enterprise/databases. |
| btrfs  | Modern, supports snapshots, pooling, checksums. |
| vfat   | FAT32, compatible across OSes. |
| ntfs   | Windows file system, read/write support in Linux via drivers. |

---

### Creating a File System

```sh
mkfs.<file_system_name> <Partition>
# Example:
mkfs.ext4 /dev/sda1
```

## Mounting File Systems

- **Mounting**: Making a file system accessible at a directory (mount point).

### Mounting Commands

- `mount <device> <mount_point>` — Mount a file system.
- `umount <mount_point>` — Unmount a file system.
- `mount -a` — Mount all in `/etc/fstab`.
- `mount -t <file_system_type> <device> <mount_point>` — Specify file system type.
- `mount -o <options> <device> <mount_point>` — Use specific options.
- `df -h` — Show mounted file systems and usage.

---

### Mounting Types

1. **Temporary Mount**
        - Use `mount` command.
        - Not persistent after reboot.
        - Example:  
            ```sh
            mount /dev/sda1 /mnt/mydisk
            ```
2. **Permanent Mount**
        - Configure in `/etc/fstab`.
        - Auto-mounts at boot.
        - Example `/etc/fstab` entry:
            ```
            /dev/sda1 /mnt/mydisk ext4 defaults 0 0
            ```
3. **Auto Mount**
        - Mounts file systems based on type.
        - Example:  
            ```sh
            mount -a
            ```

---

### Useful Commands

- `df -h` — Display mounted file systems and usage.
- `lsblk --fs` — List block devices and file system types.
- `lsblk -f` — List block devices with file system info and UUID.
- `mount -a` — Mount all in `/etc/fstab`.

## Swap Partition

- Used as virtual memory when RAM is full.
- Allows disk space to act as extra memory.
- Can be a dedicated partition or swap file.
- Improves performance and stability in Linux.
