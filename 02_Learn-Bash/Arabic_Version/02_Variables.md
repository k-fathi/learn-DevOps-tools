## 2. الـ Variables والاقتباسات (Variables and Quoting)

الـ Variables دي نقدر نعتبرها "صناديق" بنخزن فيها الداتا أو المعلومات لفترة مؤقتة وإحنا شغالين في التيرمينال أو جوه إسكربت.

### أنواع الـ Variables (Variable Types)

1. **الـ Variables المحلية (Local Variables):**
    * بتتعرف إزاي: `var_name="value"` (مهم جداً / قوي: مفيش مسافات قبل أو بعد علامة `=`).
    * الشروط الأساسية: حساسة لnnحالة الأحرف (يعني `A` غير `a`)၊ ومينفعش تبدأ برقم.
    * نطاقها: بتعيش بس جوه الجلسة الحالية. لو التيرمينال فتحت Shell تانية (Child shell) عشان تنفذ إسكربت معين، الإسكربت ده مش هيقدر يشوف الـ Variable المحلي اللي إنت عملته بره، إلا لو إنت صدرته ليه.
2. **Variables البيئة / العامة (Environment/Global Variables):**
    * دي الـ Variables اللي إنت عملتلها (Export) عشان تتشاف من أي Process أو إسكربت جوا الـ Shell دي.
    * الطريقة: `export var_name="value"`.
    * أمثلة مشهورة: `$PATH` (Paths التشغيل)، `$USER` (اسم المستخدم الحالي)، `$HOME` (Path فولدر المستخدم)، `$PWD` (Path الفولدر اللي إنت واقف فيه في اللحظة دي).

### ملفات إعدادات الـ Shell (Profile Scripts)
الـ Variables والـ Aliases (اختصارات الأوامر) اللي بتعملها بتروح لو قفلت التيرمينال. عشان تخليها ثابتة وتتحمل دايماً أول ما تفتح النظام، بنحطها في ملفات معينة:
* `/etc/profile`: ده ملف عام أو سيستم وايد (System-wide)، أي حاجة بتتحط هنا بتشتغل مع كل المستخجمين على السيستم في مرحلة اللوجن.
* `~/.bash_profile` أو `~/.bash_login` أو `~/.profile`: دي ملفات خاصة بالمستخدم لكل واحد، وبتتقري مرة واحدة بس أول ما يعمل لوجن.
* `~/.bashrc`: ده أهم ملف لينا، لأن أي تيرمينال Non-login (يعني تيرمينال جديدة بتفتحها وإنت أوريدي جوه السيستم) بتقرأ الملف ده. هنا أغلب الناس بتحط الـ Aliases وتظبيطات الألوان ومسارات الـ `PATH` الخفيفة. (بالمناسبة، الـ `~/.bash_profile` عادة بيكون جواه كود بيشغل `~/.bashrc` بشكل أوتوماتيك).

### الـ Variables الجاهزة في باش (Predefined Variables)
الباش بتوفرلك شوية Variables جاهزة عشان تسهل عليك سحب أي بيانات دخلت للإسكربت من بره (يعني إكس داتا مبعوتة للسكريبت وانت بتندهله جوه التيرمينال):
* `$0`: بيشيل اسم الإسكربت نفسه زي ما انت نديته (مثلاً `./script.sh`).
* `$#`: بيجيبلك العدد الكلي للبارامترات أو المعطيات اللي اتبعتت للإسكربت ده.
* `$*` و `$@`: الاتنين بيجيبوا "كل" البارامترات اللي اتبعتت كسترينج واحد، بس الفرق بيظهر لما تحطهم بين دابل كوتس `"$*"` و `"$@"` (زي هنشوف في الـ Loops بعدين).
* `$1, $2 ... $9`: دي البارامترات بالترتيب. يعني أول كلمة اتكتبت بعد اسم الإسكربت هتتخزن في `$1`، والتانية في `$2` وهكذا.

### الاقتباسات (Quoting & Escaping)
موضوع علامات التنصيص (Quotes) في الباش هو اللي بيحدد هل الـ Shell هتحاول تفك شفرة الكلام وتترجمه (Evaluate) ولا هتاخده زي إيه هو (Literal value):
1. **دابل كوتس أو الأقواس المزدوجة `""` (Weak Quoting):** بتسمح بتعويض الـ Variables وعمليات الاستبدال.
   * مثال: `echo "Hello $USER"` هتطبع `Hello karim`.
2. **سنجل كوتس أو الأقواس الفردية `''` (Strong Quoting):** بتاخد الكلام بالحرف الواحد زي إيه هو، وبتعطل أي مفعول لأي علامة خاصة جوه.
   * مثال: `echo 'Hello $USER'` هتطبع `Hello $USER` بالنص.
3. **البلاك سلاش `\` (Escaping):** بتلغي مفعول العلامة اللي بعدها مباشرة وبتحولها لنص عادي. 
   * لو عايز أطبع كلمة `$USER` ومخليهاش تترجم، وفي نفس الوقت عايز أستخدم الدابل كوتش لأسباب تانية، هقول: `echo "Hello \$USER"`. الباش هتشوف الباك سلاش فهتعرف إن الـ `$` هنا مجرد حرف مش علامة Variable.
