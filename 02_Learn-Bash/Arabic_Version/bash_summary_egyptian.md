# الدليل الشامل لبرمجة إسكريبتات الباش (Bash Shell Scripting): المفاهيم، التنفيذ، والأدوات المتقدمة

الدليل ده يعتبر مرجع شامل بنسبة 100% متوسع في كل تفاصيل برمجة وتكوين الـ Bash. هنغطي فيه كل حاجة من أول الهيكلة المعمارية لحد الأوامر المتقدمة في معالجة النصوص وجدولة المهام. الهدف منه انه يكون مرجع للـ Senior بيفهم فيه تفاصيل الشغل للـ Junior بشكل ودي واحترافي.

![Pipeline Architecture](pipeline_ar.png)

## 1. المعمارية: الـ Kernel، الـ Shell، والـ Terminal
عشان نفهم لينكس والـ Bash صح، لازم نفهم أيه الفرق بين التلات مكونات دول لأن دي الأساسيات.
- **الـ Kernel:** ده المحرك الأساسي أو الـ core بتاع نظام التشغيل (Operating System). هو اللي بيتحكم في الهاردوير (CPU، ميموري، وأجهزة الإدخال والإخراج) والعمليات (processes). وبيشتغل في وضع الـ kernel space أو الوضع ذات الـ Permissions العالية.
- **الـ Shell:** ده المترجم (interpreter) أو الطبقة اللي فالنص. هو الواجهة اللي بين المستخدم والـ Kernel. بياخد الأوامر اللي إنت كاتبها (human-readable) ويترجمها لتعليمات الـ Kernel يقدر يفهمها وينفذها. الـ Bash (أو Bourne-Again Shell) هو أشهر shell موجود في بيئة لينكس.
- **الـ Terminal:** ده الواجهة اللي إنت بتشوفها كمستخدم أو الـ terminal emulator (زي الـ GNOME Terminal أو Alacritty). وظيفته انه ياخد الداتا اللي بتكتبها ع الكيبورد (stdin)، يبعتها للـ Shell، بعدين يستقبل الـ Outputs سواء كانت نتايج (stdout) أو أخطاء (stderr) ويعرضهالك.

*باختصار:* إنت بتكتب في الـ **Terminal** -> الـ **Shell** بيترجم الأوامر -> الـ **Kernel** بينفذ الكلام ده على الهاردوير.

## 2. الإسكريبتات، الأوامر، وتضبيط بيئة العمل (Environment Setup)
أي إسكريبت بيتعمل بالـ Bash هو في النهاية ملف نصي (plain text) جواه شوية أوامر.

### سطر البداية (The Shebang `#!`)
أي إسكريبت لازم يبدأ بسطر الـ Shebang. السطر ده بيقول لنظام التشغيل إيه هو المترجم اللي هيتولى مهمة قراءة وتنفيذ الملف.
- `#!/usr/bin/bash`: بيدي Path الـ bash التنفيذي بشكل صريح وثابت (hardcoded).
- `#!/usr/bin/env bash`: ده بيطلب من النظام يدور بشكل ديناميكي على أول `bash` يقابله في Variable الـ `$PATH` الخاص بالمستخدم. ودي الطريقة الأفضل والموصوفة في شغل الـ CI/CD pipelines عشان تضمن إن الإسكريبت يشتغل على أي بيئة لينكس.

### أوامر أساسية لا غنى عنها للـ Shell
- `cat /etc/shells`: بيعرضلك كل الـ login shells المتوفرة والمدعومة حالياً على السيستم بتاعك.
- `echo $SHELL`: بيطبع Path الـ shell الافتراضي اللي إنت شغال بيه دلوقتي.
- `which <command/shell>`: (مثلاً `which bash`) بياخد اسم الشئ اللي بتدور عليه ويطبعلك المسار الأساسي بتاعه (absolute path) بعد ما يدور في الـ `$PATH`.
- `file <script.sh>`: بيقولك الملف ده نوعه إيه بالظبط. لو هو إسكريبت هيكتبلك إن ده "POSIX shell script, ASCII text executable".
- `chmod u+x <script.sh>` أو `chmod 700 <script.sh>`: بيدي Permissions التنفيذ (execute permissions) لصاحب الملف. من غير الـ Permissions دي، مستحيل تشغل الإسكريبت كبرنامج تنفيذي.

### طرق تشغيل الإسكريبتات
إزاي بتشغل الإسكريبت بيأثر بشكل كبير على الـ Variables وبيئة العمل (environment) اللي الإسكريبت ده بيشتغل فيها:
1. **استخدام المسار المباشر (Absolute/Relative Path)** (`./script.sh` أو `/home/user/script.sh`): الطريقة دي بتشترط وجود Permissions التنفيذ الـ `+x`. السيستم هنا بيفتح **shell فرعي جديد (new child shell)** ينفذ الإسكريبت ويقفله. أي local variables بتتعرف هنا بتتدمر وتختفي أول ما الإسكريبت يخلص ومبتتثبتش في الشاشة اللي إنت فاتحها.
2. **استخدام المترجم بشكل صريح (Interpreter Explicitly)** (`bash script.sh` أو `sh script.sh`): مبتطلبش Permissions التنفيذ. ومباشرةً بتفتح برضوا **new child shell**.
3. **التشغيل بالمصدر (Sourcing)** (`source script.sh` أو `. script.sh`): الطريقة دي مبتطلبش Permissions التنفيذ ومبتفتحش shell جديد!. دي بتنفذ الأوامر جوا **نفس الـ shell الافتراضي الحالي**. يعني أي functions أو variables إتعملت في الإسكريبت هتفضل معاك مكملة في الـ terminal session اللي إنت فاتحه حتى بعد ما الإسكريبت يخلص.

#### تعديل الـ Variable `$PATH`
علشان تقدر تشغل الإسكريبت باسمه من أي مكان في السيستم من غير ما تكتب مساره، لازم تضيف المسار الرئيسي للإسكريبت جوه الـ Variable `$PATH`:
```bash
export PATH=$PATH:/path/to/your/script_directory
```
عشان تثبت التعديل ده حتى لو عملت restart للسيستم، لازم تاخد نفس السطر وتحطه في ملف `~/.bashrc` أو `~/.bash_profile`.

## 3. الـ Variables وعلامات التنصيص (Quoting)
الـ Variables دي مكان في الميموري بنخزن فيه البيانات مؤقتاً أثناء عمل السيستم أو تنفيذ الإسكريبت.

### أنواع الـ Variables (Variable Types)
1. **Local Variables:** بتتعرف بالشكل ده `var_name="value"` (لازم ومهم جداً / قوي ما يكونش فيه مسافات حوالين الـ `=`). أسامي الـ Variables بتكون حساسة لحالة الأحرف (case-sensitive) ومينفعش تبدأ برقم. الـ variables دي بتعيش بس جوة الـ shell اللي احنا شغالين فيه. لو الإسكريبت فتح child shell، مش هيشوف أي local variable متعرف بره.
2. **Environment/Global Variables:** دي Variables بتتعمل لها تصدير (export) لأي Processes أبناء بتفتح. زي `export var_name="value"`. من الأمثلة الافتراضية بتاعت السيستم `$PATH` و `$USER` و `$HOME` و `$PWD`. تقدر تعرضهم بأمر `env` أو تمسحهم بأمر `unset var_name`. الـ Variables دي بتنتهي تماماً أول ما السيرفر يتعمله restart.
3. **Variables دائمة على مستوى السيستم (System-Wide Persistent Variables):** دي Variables بتتحط في ملفات التكوين الأساسية زي `/etc/profile` أو `/etc/environment`. الـ Variables دي بتأثر في كل المستخدمين وبتفضل موجودة بعد الـ restart. أي تعديل في `/etc/profile` بيحتاج إنك تسجل خروج وتدخل تاني أو تعمله sourcing.

### ملفات التكوين للـ Bash (Configuration Files)
- `/etc/profile`: على مستوى السيستم بالكامل، بيتحط فيها الـ environment variables الخاصة بالـ login shells لكل الـ users.
- `~/.bashrc`: خاصة بالمستخدم الحالي بس (user-specific). بيتحط فيها اختصارات الأوامر (aliases) والإسكريبتات والدوال اللي بتشتغل بمجرد فتح terminal جديد.
- `/etc/bashrc`: على مستوى السيستم بالكامل للـ non-login shells.
- `/etc/environment`: ملف للسيستم ككل، بياخد Variables ثابتة بس (static string) ومينفعش يتكتب فيه أوامر برمجية. تأثيره دايم ومبيتمسحش بـ reboot.

### Variables خاصة مبنية مسبقاً (Predefined/Special Variables)
- `$0`: بيرجع اسم الإسكريبت اللي شغال.
- `$#`: عدد الـ arguments أو الـ Parameters اللي اتبعتت للإسكريبت.
- من `$1` لحد `$9`: دي الـ Parameters المترتبة اللي اتبعتت.
- `$!`: المعرف القياسي (Process ID - PID) لأخر أمر اشتغل في الخلفية عن طريق الـ `&`.
- `$?`: كود Exit Status بتاع أخر أمر اتنفذ.
- `$*`: بيلم كل الـ arguments في نص واحد (string) (لو كان محطوط بين علامتين " ").
- `$@`: بيلم كل الـ arguments بس بيحطهم كأنهم List أو كذا string مستقل. وإحنا دايماً بنفضل بنستخدم `"$@"` لما نيجي نمر على قيم الـ arguments في حلقة دوران (loop) علشان نحافظ على المسافات اللي بين الـ arguments.

### قواعد التنصيص (Quoting Rules)
- **Double Quotes (`" "`):** دي (weak quoting). بتاخد النص كـ string بس بتترجم أي Variable `var$` جواه وتغيره لقيمته. وبتشغل الـ command substitution وتستجيب للتعليقات زي `n\`.
- **Single Quotes (`' '`):** دي (strong quoting). بتاخد النص كأنه نص بحت. يعني `USER$` هتطبعلك `USER$` زي إيه هي مش هتطبعلك اسم المستخدم.
- **Escape Character (`\`):** ببتستخدم عشان تلغي وظيفة الحرف اللي واخد special meaning بعدها مباشرة. مثلاً `USER$\` بتطبع `USER$`.

## 4. العمليات الحسابية (Arithmetic Operations)
الـ Bash مافيهوش قدرة إنه يعمل Processes حسابية بشكل رئيسي بدون استخدام syntax معين للاستدعاء. والعمليات ذات الأرقام العشرية (Floating-point) مبتشتغلش في الـ Bash الطبيعي أبداً، الباش بيفهم أرقام صحيحة بس (integers).

### الحسابات المعتمدة للأرقام الصحيحة (Integers)
- `$((expression))` أو `((expression))`: الطريقة اللي ببنستخدمها في عالم الـ POSIX القياسي حالياً. كمثال: `result=$((5 + 3))`. كمان ببنستخدم الأقواس المزدوجة `(( ))` لعمل الاختبارات المنطقية مقارنة بين أرقام (مثلاً `if (( a > b ))`).
- `let`: أسلوب قديم في الباش. كمثال `let result=5+3` لكن لازم متسيبش أي مسافات.
- `$[expression]`: كمان ده أسلوب قديم وغير محبذ.
- `expr`: أمر خارجي قديم. مثلاً `result=$(expr 10 - 2)`. هنا المسافات بين الأرقام والعلامات ضرورية ومهمة جداً / قوي! ومع الضرب لازم نعمل escape بالعلامة: `expr 5 \* 3`.

### العمليات اللي بتحتاج لأرقام عشرية (Floating-Point) عن طريق أداة `bc`
الطريقة الوحيدة عشان نطلع نواتج بكسور عائمة، إننا نبعت العملية (pipe) للحاسبة `bc`:
```bash
result=$(echo "scale=2; 5 / 2" | bc)  # الـ Outputs: 2.50
result=$(echo "2.5 * 4.1" | bc)       # الـ Outputs: 10.25
```

## 5. استبدال الأوامر (Command Substitution) وInputs المستخدم (User Input)
### Command Substitution
ببنستخدمها عشان ننفذ أمر في الخلفية، ونخزن الـ output اللي بيطلعه منه جوا Variable ثاني.
- Syntax مفضل: `var=$(command)` (الطريقة دي حديثة، وممكن جواها تعمل أوامر مستعينة ببعضها بكل سهولة).
- Syntax قديمة: `var=\`command\`` (باستخدام الـ backticks مش مفضلة عشان صعبة القراءة).

### إخراج الداتا وطلبها (`echo` و `read`)
- `echo -n`: بتمنع إن السيستم يكسر السطر ويعمل سطر جديد (newline) في الاخر.
- `echo -e`: بتفعل الخصائص بتاعت علامة `\` زي مسافة الطاب `t\` و سطر جديد `n\`.
- `read`: بتاخد داتا من المستخدم أو الـ (stdin).
  - `read var1 var2`: لو كتبت جملة، هتحفظ اول كلمة لـ `var1` وتدي الباقي لـ `var2`.
  - `read -p "Prompt: " var`: بتظهرلك الرسالة الأول وبعدها توقف السكريبت مستنية إنك تكتب الرد.
  - `read -s`: الطريقة دي بتمنع الكلمة اللي بتشتكتبها تبان (silent mode) وببنستخدمها لدخول الباسورد.
  - `read -t 5`: ده timeout. لو محطيتش حاجة ف مدة 5 ثواني الأمر هيفصل وميستناش أكتر من كده.

### الـ Brace Expansion `{}`
طريقة سريعة وممتازة كترت استخدامها في الباش لإنشاء أنماط ومجموعات وحروف.
- `echo {1..5}` -> هتطبع `1 2 3 4 5`
- `echo file_{A..C}.txt` -> هتطبع `file_A.txt file_B.txt file_C.txt`
- `echo {10..0..-2}` -> هتطبع من 10 لحد صفر بانها تنقص رقمين كل مرة `10 8 6 4 2 0`

## 6. Paths الإخراج (Streams) وExit Status
أي أمر في لينكس بيشتغل، بيسيب معاه كود كحالة للخروج (Return code) بيبدأ من صفر لحد 255.
- **`0`**: العملية اشتغلت بنجاح ومفيش خطأ.
- **`1`**: العملية فشلت أو فيها ايرور عام.
- **`2`**: الأمر اتكب بطريقة غلط أو استخدمت flags مش بتاعة الأمر ده.
- **`126`**: الأمر موجود لكن معندكش Permissions أو إنه مش واخد permission `+x` إنه يشتغل كسكريبت (Permission denied).
- **`127`**: الأمر أصلاً مش موجود في السيستم وماحدش عارف المسار بتاعه فين (Command not found).
- **`130`**: السكريبت وقف بالعافية عن طريق إن الـ user داس `Ctrl+C` أو الـ (SIGINT).

بنشوف قيمة الـ Status دي عن طريق Variable `$?`. وتقدر إنت جوا السكريبت بتاعك تعمل `exit <رقم>` عشان تقفل الشغل كله بطريقة معينة.

### الأنابيب الـ Pipelines ومسارات البيانات
زي ما الباش قسم الشوغل، أي process فيه ليها 3 ملفات أو Paths أساسية شغالة:
1. **0 (Standard Input - stdin):** الـ Inputs اللي بياخدها البرنامج (زي الكيبورد بتاعك).
2. **1 (Standard Output - stdout):** الـ Outputs الصح ولون الطباعة الأساسية للبيانات الناجحة.
3. **2 (Standard Error - stderr):** شاشة عرض الأخطاء والـ Exception logs.

**توجيه المسار (Redirections):**
- `> file`: بياخد النواتج بتاع الـ stdout ويمسح الملف لو كان موجود و يكتب النتيجة مكانه.
- `>> file`: بيكمل على نفس الملف من غير ما يمسح البيانات القديمة.
- `2> error.log`: بياخد أي error طلع (stderr) ويرمية ف لوج.
- `&> full.log` أو `> full.log 2>&1`: الطريقة دي بتخلي الناجح والـ Error يترموا في نفس الملف.

**الأنابيب (`|` - Pipelines):**
تخيل الماسورة اللي بتنقل الماية، البايب ده بياخد الـ stdout بتاع الجزء اللي عالشمال ويدخله كInputs (stdin) في الجزء اللي عاليمين.
مثال: `cat /etc/passwd | grep "/bin/bash" | wc -l` (يفتح الملف -> يصفي بس السطور اللي فيها بش -> يعد كام سطر اتصفي بالظبط).

## 7. الجمل الشرطية (`if` و `case`)

![Logic Flowchart](logic_ar.png)

### الجملة الشرطية `if`
بناءً على النواتج الخاصة بالعمليات الـ if بتمشي أو لا.
```bash
if [[ condition ]]; then
    # السطور دي هتتنفذ لو الشرط الأول تحقق
elif [[ condition2 ]]; then
    # لو الشرط الأول باظ, هيجرب هنا. ولو حقق بينفذ دي بس
else
    # لو كله باظ ده الاختيار الافتراضي الحتمي
fi
```

### الصيغة بتاعت Processes الاختبار والتأكد
1. **`[ ]` (Processes اختبار الـ POSIX القديمة):** بتقدر بتستخدمها لمقارنة الـ strings والملفات. بس ممكن تحصل مشكلة مع تقسيم الكلمات لو الـ Variable محطوط فيه فراغات (Word Splitting)، ولازم تخليك حريص وتحط الـ variables دي دايماً بين علامات " ".
2. **`[[ ]]` (الأسلوب المفضل والأحسن في Bash Extended Test):** دي الطريقة الأنضف والأكثر تقبلاً. مبتعملش word splitting وتطرد أخطاء المسافة. وفيها دعم للبحث المنطقي بتاع الـ Regex زي `=~` وتدعم بشكل أساسي إنك تكتب جواها `&&` و `||` من دون ما تعمل errors.
3. **`(( ))` (حساب رياضي):** الـ method ده بيُستخدم بس عشان تقارن الأرقام ببعض.

### الـ File Operators
- `-e file`: بيتأكد إن الفايل ده موجود أيا كان نوعه إيه.
- `-f file`: ده ملف فايل عادي مش فولدر مثلاً وموجود فعلًا.
- `-d dir`: بيتأكد إن ده فولدر.
- `-r file`, `-w file`, `-x file`: بيشوف هل الفايل ده انت ليك حق تقرأ منه ولا تكتب فيه ولا تشغله (Permissions).
- `-s file`: موجود ومش فاضي. يعني مساحته أكبر من صفر.
- `-z string`: للـ String... لو كان الاسترينج فاضي كلياً (length = 0)، بيرجع الصح.
- `-n string`: لو الـ String كان مليان وفي مسافات مش فاضي.

### العلامات الرقمية والنصية (Operators)
- **للنصوص (`[[ ]]`):** بتحتوي على `=`, `==`, `!=`, `<`, `>`.
- **للأرقام (`[ ]` أو `[[ ]]`):** لازم بتستخدم المقارنات المعقدة بالكتابة زي: `-eq` (يتباع بـ '='), `-ne` (نفي), `-gt` (أكبر من), `-lt` (أصغر من), `-ge` (أكبر أو يساوي), `-le` (أصغر أو يساوي).
- **للأرقام في (`(( ))`):** بتقدر بتستخدم كل العمليات الرياضية بسهولة `>, <, ==, !=, >=, <=`.

### استخدام `case`
طريقة أقصر، وأريح للعين وأسهل بدل سلسلة معقدة من `if-else` مع Variable واحد ومحتاج تبني عليه قرارات سريعة كتير مع الـ pattern matching.
```bash
case $var in
    pattern1|pattern2)
        echo "هنطبع هنا لو الـ Variable مشي مع الأول أو التاني"
        ;;
    [A-Z]*)
        echo "الكلام بيبدأ بحرف كابيتال"
        ;;
    *)
        echo "لو كله باظ يبقى دي حالة الافتراضي (Catch-All)"
        ;;
esac
```

## 8. الـ Loops والدوران (Loops المتمثلىة في `for, while, until, select`)
### جمل الـ `for`
بنمر بيها على مشته كلمات وراء بعض، مجموعات، List of strings أو List من الأرقام.
- **Iteration list:** `for item in apple banana orange; do echo $item; done`
- **Brace Expansion list:** `for i in {1..10}; do echo $i; done`
- **على طريقة لغة السي C-Style اللى احنا كلنا عارفينها:** `for ((i=0; i<10; i++)); do echo $i; done`

### اوامر الدوران عن طريق `while` و `until`
- **`while`**: اللوب شغال طالما Condition معين **بيطلع بـ صح (True)**. بيفضل شغال لغاية متبقا النتيجة (False). (زي: `while [[ $count -lt 5 ]]`)
- **`until`**: اللوب شغال طالما Condition ده **غلط (False)**. ومجرد ميعكس حالته ويكون True بيقف علطول. (زي: `until [[ -f /tmp/ready.txt ]]`)

### القراءة بالسطور
أنجح، وأأمن وأكتر طريقة بيفضلوها في قراءة لملفات سطر بسطر في الباش من عشان منتفقدش الـ whitespace أو رموز الهروب الـ Backspaces:
```bash
while IFS= read -r line; do
    echo "Processing: $line"
done < input.txt
```
- `IFS=`: بيمنع إننا نمسح المسافات اليمين أو الشمال من الفايل.
- `-r`: بتخلي read ماتحولش الرموز اللي معمولة بالتهريب `\` لمعاني مختلفة، فتخلي الفايل يتقرا زي إيه هوا بالظبط RAW.

### إدارة حركات اللوب (`break` أو `continue`)
- `continue`: بيتجاهل اللي جي من اللوب دي ويرجع للدورة اللي بعدها علشان يتختبر Condition تاني.
- `break`: بتلغي اللوبس نهائيا وتطلع بره البلوك كله.
- `break 2`: لو فيه تداخل بين اللوبات وبعض (Nested Loops) زي اتنين جوا بعضك الطريقة دي بتكسر الاتنين في نفس اللحظة وتخرج برة خالص.

### قائمة التحديد `select` Loop
بتعمل CLI Menu إحترافية وسهلة وسريعة للاستخدام. بتديلها عناصر بترقمهم هيا وتستنى المدخل للمستخدم وبنحط اختياره في variable جاهز للشغل.
```bash
select os in Linux Windows macOS Quit; do
    if [[ $os == "Quit" ]]; then
        break
    else
        echo "إنت اختارت يا فندم $os"
    fi
done
```

## 9. بناء الداتا في الـ Arrays (Array Data Structures)
هي Arrays List بتحجز كذا عنصر جوا اسم واحد Variable بس يكون متسلسل.

- **التعريف Initialization:** `my_array=(apple banana cherry)`
- **طلب Element معين:** `${my_array[0]}` -> الرد هنا هيكون `apple`. (البدأ بيبقا Index متعارف بـ صفر زي العادة).
- **طباعة الـ list كلها:** `${my_array[@]}`. (يُفضل استخدام علامتين تنصيص `"${my_array[@]}"` علشان نحمي المسافات بين المكونات).
- **قياس طول الحاجة Count:** `${#my_array[@]}` ده بيدي عدد الـ Elements المتوفرة ف الـ Array. و `${#my_array[0]}` بترجع عدد الحروف اللي بتكمن Element الأولى وهي "apple".
- **عايز اطبع أرقام الفهارس Keys بس:** استخدم `${!my_array[@]}` .
- **لإضافة عنصر Append:** `my_array+=(grape)`.
- **عشان تشيل Data:** `unset my_array[1]` هيمسحلك التانيه والمكان ده حيفضل مستثني لكن الباقي شغالين لأن الـ Arrays في System الباش بطبيعتها متفرقة (sparse).
- **عشان تعدي عليها في لوب (Standard Iterate Properley):**
  ```bash
  for item in "${my_array[@]}"; do
      echo "$item"
  done
  ```

## 10. التحكم المتقدم في كود السكريبت (`shift` والدوال Functions)

### الإزاحة الموضعية بأمر `shift`
أمر `shift` بينقل الباراميترز خطوة ناحية الشمال. وده بيعني إن `$2` بتبقا `$1` و `$3` بتبقا `$2`, واللى كانت في `$1` أساساً اتمسحت للأبد! بنستعمل الميزه دي للتعامل مع الـ Variables الكتير ديناميكاً في الـ flags والاوامر المطولة زى `--force` أو `--path` جوا while loop. ال أمر `shift 2` بينقلهم ياعني مرتبتين في خطوة واحدة.

### الـ Functions في الـ Bash
بتعمل بيهم Functions بحيث انك تكرر اوامر من خلال اسم، ده بيساهم ف نظافة الكود وإعادة استخدامه.
```bash
# أفضل Syntax للتعريف في نظام ال POSIX
my_function() {
    # لازم تكون واخد بالك انك تحط Variables بصورة Local عشان ماتبوظش ال Script ككل ومتضربلكش حاجه من بره.
    local var1="$1"
    local var2="$2"
    echo "دول ارطو مستقلين ومابيتأثروش بمكان ال Arguments الكبار للـسكيبرت: $var1 $var2"
    
    # عشان تفهم كويس في الباش مشبنستعمل أمر Return غير لطباعه ال Exit Code بس (من 0 لـ 255)
    # لاكن علشان ارجع قيمة string ملموسة ببنستخدم الطباعه ال Echo
}

# بنستدعي الفانكشن منغير أي اقواس
result=$(my_function "First" "Second")
```
في الشركات والمشاريع الكبيرة في Linux بنلم شوية Functions كبيرة ونحطهم كلهم في file مستقل، ولما نكون عاوزينهم ف فايل Script العمل الرئيسي بندي أمر `source libraries/utils.sh`. بكده الاوامر دي هترمي الـ Functions بالكامل في الـ namespace بتاعة السكريبت، وبنقدر ننده عليها على طول للـ Execution.

## 11. إستخدام الـ Regular Expressions (Regex) بقوه الباش
المعرف كـ "أنماط الـ Regular Expressions (Regex)" عبارة عن أداة بحث جبارة لفلترة وعمل validations عن طريق وضع رموز Pattern معينه تفلتر الحروف. بيستخدم جوا الاختبارات باستخدام علامة "=~" أو ضمن Functions تحليل الداتا زى `sed`، و `awk`، و `grep`.

- `^`: معناها إن الشرط ده يبقا من البداية حصرياً بداية الكلمة أو السطر.
- `$`: معناها حصرياً في أخر نقطة و قفلة السطر String.
- `.`: بيماثل أى شكل، أى مسافة أو حرف أو رقم. هو حرف وحيد مبهم!
- `[abc]` / `[a-z]`: صندوق للطبقة، ده بيطابق اى حرف واحد بس موجود جوه الصندوق اللي احنا محددينه ده.
- `[^abc]`: النفي للطبقة يعني أى حرف تانى في الدنيا موجود في الكلمة بشرط ميكونش من الأنماط الداخليه جوا المتاريس دي!
- `*`: بيشاور ع اللي قبله ويقوله إطبعلي صفر مرة أو لا نهائي من الحروف... `a*` هتبقا قشطه للنتيجة دي ``, `a`, `aa`, `aaa` الخ.
- `+`: من الـ Extended Regex المتقدمة - بطابق واحد أو اكتر! كمثال `[0-9]+` يعني بيمشى مع أي عدد رقمي زي ال `5` ، و ال `5000`.
- `?`: دي كمان Extended Regex. بيطابق حاجة من الاتنين صفر أو مرة واحده وهو Optional اختياري. يعني `colou?r` بتمشى مع الأمرين `color` بالاضافة الى `colour`.
- `{n}`: Extended Regex. لازم تتأكد من وجود Pattern متكرر بعدد N. فمثلا `[0-9]{3}` يعني بتدور بس على عدد متكون بالتخصيص من (ثلاث أرقام) بالظبط زي 435.

مثال للبحث المباشر عن طريق السكريبت بالباش للـ Regex:
```bash
email="karim@devops.com"
if [[ "$email" =~ ^[A-Za-z0-9._%+-]+@[A-Za-z0-9.-]+\.[A-Za-z]{2,}$ ]]; then
    echo "الايميل بتاعك ده صحيح و Valid مليون ف المية!"
fi
```

## 12. حقيبة المعالجة للـ Data Processing Toolkit بالأدوات: SED و AWK
في وقت الباش بيجيلك بالعمليات والحلقات بتاعته واللوبز بيعطل، بيجي الـ Linux Standard انه معتمد دايماً في تحليل الداتا النصية بقوة على ادوات مخصصة Stream Editors متبرمجة بلغة الـ C، وأسرع بكثير جداً / قوي، وهي `sed` والـ `awk`.

### معالجة الداتا بإستخدام (The Stream Editor `sed`)
ده مختص بعمل Modification والتصحيحات والاستبدال في الـ strings والمستندات في وضع غير تفاعلي Non-interactive مستخدماً قوة وخفيات الـ Regex.
- **التخطيط البنائي لعمل أمر:** `sed 's/هنا_نمط_البحث/هنا_نص_التبديل/الفلاج_مكمل' file.txt`
- **التغير أو الاستبدال الشامل Global (g):** الدائم انه مش بيغير الا اول حاجة تتقابله بس. لو بتعمل `sed 's/foo/bar/g' file.txt` ال g بترغمه يغير كل الكلمات المقابلة بطول ونهاية السطر مش أول واحدة بالـ Shell!
- **الحفظ مكان العمل بصمت (-i):** عشان بطبيعة الحال sed بيطبعلك ع الـ screen و مبيرضاش يلعب فى الفايل الأساسي... بتكتب `-i` هيقوم عامل Save مباشرة في نفس الملف.
- **التطرق للأرقام المحدده Line Addressing:** `sed '2,5s/foo/bar/'` الأسطر من الثانى للخامس بس هما اللي الامر بيطبق عليهم. ورمز `$` بيعبر دايما عن النهاية، فنقول `2,$` يبقى لحد أخر الملف.
- **المسح والإسقاط (d):** هنا معاه السيف بيمسح. زي الحذف بالكامل للأسطر اللى بتبدأ بالشباك (`#`) يعني هي الكومنتس `sed '/^#/d' file.conf` ، أو حتى الحذف للأسطر الفاضيه نهائي بالامر `sed '/^$/d' file.txt`.
- **موافقة النواتج او كتم الـ Outputs الافتراضية للطباعة (-n) والـ (p):** بنجمعهم كدا `sed -n 's/foo/bar/p'` وبيؤدي ان السكريبت بيسكت خالص وبيطلع بسطط ال Record اللي حصله substitution وغير كده لأ.

### أداوت تحليل اللغة الرهيب للـ Patterns اللي اسمه Pattern Scanning & Processing  (`awk`)
ده مش أمر. دي لغة برمجية كاملة تعتبر "Turing-complete" مختصة بهندسة وتفنيد و Extraction للبيانات المعقدة والمتسجلة أعمدة Columnar text. لما بتبعت سطر (بحاله) لـ AWK بيطلق عليه في ال System عندهم انه "Record"، وكل كلمة جوا العمود مفصولة عن بعض بطبيعة الحال بإسم "Field".
- **إزايية معالجة أمر التنفيذ Execution Architecture:** `awk 'BEGIN {Initialization} /Pattern Match/ {Action on each line} END {Final Execution}' file.txt`
- **شرح الـ Variables المهمة اللي بتستعمل Variables Usage:**
  - `NR`: ده الـ Record الحالي ياعني مجرد ترقيم السطر (Number of Records).
  - `NF`: ده رقم ياعني الـ Line الحالي فيه عدد كذا حقل Column field. فاهمني؟ Number of Fields!
  - `$0`: رقم المعامل صفر في ال Range بيطبعلك السطر الـ Raw كله سليم على بعضه.
  - `$1, $2, $3...`: البوينتر Pointer اللي بيوديك علي المكان الاول او التاني او التالت بالتحديد للحقل Field المختار.
  - `FS`: ده المفرق أو الفاصل الافتراضي، الديفولت Default هو حبة 'المسافات الفاضية'. لكن لو عايزها ":" زي ال Passwd, نقدر نغيره ב `BEGIN {FS=":"}` أو في الامر نفسه بـ الفلاج `-F :`.
  - `OFS`: بيعمل Separation في Outputs شاشة الاخراج بتاعتك، لتعديل طريقة عرض الإخراج.
- **إمكانيات ریاضیة مهولة:** اللغة Awk بتدعم Processes الديسيمال Fraction والكسور ببراعة رهيبة عن الباش.. يعني تجمع مثلا `awk '{ sum += $3 } END { print "Total:", sum }' file.txt`.
- **ترشيح بالمنطق Logic Filters:** تقدر تقول للمحرك Awk فكك من الشغل إلا في Case منطقية بتعمل Mapping:
  - `awk -F : '$3 >= 1000 { print $1 }' /etc/passwd`: لو لقيت الفيلد الـ 3 اللى هوا (UID) يكون ال الراعمل تاعه المبدأ `>= 1000` .. أطبعلي الفيلد التاني `$1` اللى هوا اليوزر.

## 13. أتمتة وجدولة مهام السيستم (System Task Automation and Scheduling)
لينكس بطبعه بيحبذ الأتمتة وانه يقوم بالروتين المُمل والادارة، وده بيفعله عن طريق Back-end daemons بتشتغل فى السيستم natively واهمهم على الاطلاق (`atd` و `crond`).

### التنفيذ مرة واحدة بس ياعني One-Time Execution (`at`)
بتجبر الجهاز يعمل Execute بقرار، في وقت معين ف الـ Future مره واحده بس وينساه.
- `at 23:00 -f launch.sh`: بقول للسيستم يعمل Load لـ `launch.sh` النهاردة بليل الساعة 11.
- `at now + 2 hours`: هدخلك جوا shell فرعي واقررلك تعمل شوية كاتبة سكريبتات، الحاجات دي هتشغلها بالظبط بعد ساعتين من أرقام الساعة دلوقتي.
- `atq`: ده أمر يخليك تعرض كل الاوامر اللي مرصوصة ولسه متنفذتش فى ال List بتاع المهام لـ User ال at.
- `atrm <ID>`: بتكنسل وتحذف الجدول بـ وضع ID اللي مسكه الأمر جوا قائمة ال List.

### الجدولة الحتمية المتكررة الـ Sub-System Recurring Chronology (`crontab`)
في قلب السيستم الـ Cron الديمون ده صحي كل دقيقة واحدة، بيعدي يسأل في جداول ال Configuration، "يا جماعة حد حاطيت Trigger لتاسكات اعيدها تاني؟" وبيروح ينفذ التكرار ده بطريقة آلية للسكريبت.

**شغل المستخدم العادي الـ User Configurations باستخدام الـ (`crontab -e`):**
بتفتح Table عشان تنفذ شغل ال التاسكس الدورية من خلال User Authority للمستخدم الحالي اللي مشغله حالياً.
- النجمات Asterisk بتترتب بالشكل اللى جاي Default وهي: `الدقيقة Minute (0-59)` و `الساعة Hour (0-23)` ثم `يوم من الشهر Day/Month (1-31)` তারপর `الشهر نفسه Month (1-12)` وفي الأخر الـ `يوم الأسبوع Day/Week (0-7, حيت الصفر او ال 7 دا الأحد)`.
- `0 12 * * 1-5 /path/script.sh`: تنفذ الأمر الساعة 12:00 الظهر دايماً لو انهاردا يوافق يوم من الإثنين لحد يوم الجمعة بس.
- `* * * * * command`: ده Trigger مبيوقفش، هيقوم يشتغل كل دقيقة واحدة متتالية كل يوم، كل شهر على طوووووول!
- اوامر مختصرات الماكرو Macro implementations: زى علامة ال `@reboot` عشان نشتغل أول مالـ Server الـ Reboot يتم. ومثلهم `@daily` اللى تعني نص الليل Midnight، وبرضه `@hourly`.

**المحددات المتقدمة للمديرين او Administration Constraint Files الـ (`/etc/crontab`):**
الملف اللي موجود في Path `/etc/crontab` ده فايل للـ Root والمديرين بيحتوي على parameter مخصوص كزيادة، علشان بيقدروا يحددوا مين هو الـ user اللي هيخلي الأمر ده Run تحته. مثال: (`0 2 * * * root /bin/sync`)... ده بيرن للـ root.
ببتستخدم كمان الـ السيستم ملفات Directory رئيسية زى `/etc/cron.daily/`، `/etc/cron.weekly/`. وتقدر حضرتك تحدف جواها Script Programs تنفيذية executable والـ Daemon ال Cron بيكتشفهم آلياً ويرن عليهم بالوقت بتاعهم كديفولت بدون وجع دماغ ال Setup فى فايلات الجدولة.
