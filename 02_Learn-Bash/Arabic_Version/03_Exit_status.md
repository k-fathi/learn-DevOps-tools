## 3. أمر وExit Status (Exit Command and Exit Status)

في لغة الباش، أي أمر بتكتبه أو إسكربت بتشغله بيبعت رسالة للنظام في النهاية بيقوله "أنا خلصت شغل" وبيدي كود أو راعمل يسموه (Exit Status) عشان يوضح هو خلص بنجاح ولا حصلت مشكلة.
- **تم بنجاح (Successful execution)**: بيرجع صفر `0`.
- **حصل خطأ (Failure)**: بيرجع أي رقم تاني غير الصفر `≠ 0`.

### إزاي تعرف Exit Status؟
بتقدر تشوف Exit Status بتاع **آخر أمر اتنفذ حالاً** عن طريق Variable خاص اسمه `$?`. 
مثلاً لو كتبت:
```bash
echo $?
```
رقم `0` معناه إن الأمر اللي قبله تمام، أي رقم تاني معناه إن فيه مشكلة.

### أشهر أكواد الخروج (Common Exit Codes)
ودي قايمة بأهم الأرقام اللي ممكن تشوفها ومعانيها:

| كود الخروج | المعنى |
|-----------|----------------------------------------------|
| `0`       | الأمر أو الإسكربت خلص شغله بنجاح تام. |
| `1`       | خطأ عام أو غير محدد (General error). |
| `2`       | استخدام غلط للأمر، أو كتبت بارامترات مش مظبوطة (Invalid argument). |
| `126`     | الأمر موجود بس معندكش صلاحية تنفيذه (Permission Denied). |
| `127`     | الأمر اصلاً مش موجود أو مبقاش مسطب (Command not found). |
| `130`     | الإسكربت وقف فجأة لإنك دوست `Ctrl+C` (Keyboard Interrupt). |
| `255`     | كود بره النطاق (Out of range)، لأن أكواد الخروج بتوصل لحد 255 بس. |

### ليه بنهتم بExit Status أصلاً؟
Exit Status دي أساسية جداً / قوي عشان:
1. **الديابجينج (Debugging)**: بتعرفك بسرعة فين المشكلة في سلسلة الأوامر بتاعتك.
2. **المنطق والبرمجة (Scripting Logic)**: بنبني عليها الشروط زي الـ `if` بحيث نأمر الإسكربت إنه يعمل كذا "لو" الأمر الفلاني نجح، أو كذا "لو" فشل.

مثال باستخدام `if`:
```bash
if command; then
    echo "الأمر نجح."
else
    echo "الأمر فشل وExit Status بتاعته هي $?."
fi
```
> **نصيحة للمحترفين:** دايماً اعمل حساب أكواد الخروج (Error handling) في إسكربتاتك خصوصاً اللي هتروح للبرودكشن، عشان متسيبش السيستم يضرب منك من غير ما تعرف السبب.

### أمر الـ `exit` نفسه
الأمر ده ببنستخدمه عشان ننهي إسكربت بالكامل في اللحظة دي ومكملش قراية، أو نطلع بره سيرشن التيرمينال. والأهم إننا نقدر نحدد بيه إحنا هنخرج وكود الخروج بتاعنا إيه.

#### الصيغة (Syntax):
```bash
exit [n]
```
حيث `n` هو راعمل نختاره. لو مكتبناش رقم، الإسكربت هيخرج بنفس حالة خروج آخر أمر اتنفذ جواه.

#### استخداماته (Use Cases):
1. **قفل الإسكربت بنجاح**:
    ```bash
    #!/bin/bash
    echo "ده مجرد سكريبت."
    exit 0
    ```
    السكريبت هيخلص ويرجع `0` يعني تمام.

2. **التعامل مع الأخطاء (Error Handling)**:
    ```bash
    if [ ! -f "/path/to/file" ]; then
         echo "الملف مش موجود!"
         exit 1
    fi
    ```
    لو الملف مش موجود، السكريبت هيطبع رسالة ويقفل فوراً ويرجع كود `1` دليل على الفشل، ومش هيكمل قراية باقي السطور.

> الخلاصة: أمر `exit` مش بس بيوقف الـ `if` ستيتمنت أو الـ `loop`، ده بيوقف **الإسكربت كله** وبيطلع براه.
