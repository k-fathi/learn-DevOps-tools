## 1. الإسكربتات، الأوامر، وتجهيز بيئة العمل (Scripts, Commands, & Environment Setup)

الإسكربت باختصار هو ملف نصي عادي جداً / قوي متجمع جواه شوية أوامر، بدل ما تكتبهم أمر أمر بإيدك على التيرمينال أو الـ Terminal، بتكتبهم كلهم ورا بعض وتحفظهم، وتخلي الـ Shell تشغلهم مرة واحدة بالترتيب اللي إنت كاتبه.

### علامة التعريف أو (The Shebang `#!`)
أول سطر في أي إسكربت باش بيسموه (Shebang) وهو مهم جداً / قوي لإن نظام التشغيل بيبص عليه عشان يعرف إيه هي الـ Shell اللي هتستلم الملف ده وتشغله:
- `#!/usr/bin/bash`: هنا إنت بتثبت Path الـ Shell. يعني بتقول للنظام "أنا متأكد 100% أن الـ bash موجود في المسار ده، استخدمه". 
- `#!/usr/bin/env bash`: الطريقة دي أذكى بكتير ومفضلة في الشغل (Best Practice). لأنها بتدور على نسخة الـ bash أوتوماتيك جوا Variable المسارات (`$PATH`) بتاعك حتى لو مكانه Variable من جهاز للتاني. ده بيضمن إن الإسكربت بتاعك هيشتغل بدون مشاكل على أنظمة وأجهزة تانية (Cross-platform compatibility).

### أوامر الـ Shell الأساسية
دي شوية أدوات لاي غنى عنها عشان تعرف إنت شغال بShell إيه وفين:
* `cat /etc/shells`: الأمر ده بيطلعلك قايمة بكل أنواع الصدف المتاحة والمتسطبة على جهازك حالياً اللي تقدر تعمل بيهم تسجيل دخول (Login shells).
* `echo $SHELL`: ده بيطبعلك Path الـ Shell الافتراضية اللي شغالة معاك دلوقتي. (خد بالك دي مش دايماً هي الـ Shell اللي إنت واقف فيها فعلياً لو قمت بتغييرها بشكل مؤقت، دي الـ Shell الأساسية اللي اليوزر مربوط بيها).
* `which <command/shell>` (أو `which bash`): أداة خطيرة بتقولك المسار النهائي للأمر التنفيذي ده موجود فين بالظبط جوه ملفات النظام بناءً على الـ `$PATH` الخاص بيك.
* `file <script.sh>`: أمر لذيذ بيقولك نوع الملف إيه. لو طبّقته على إسكربت باش حيطلعلك رسالة زى "POSIX shell script, ASCII text executable". وده معناه إنه ملف نصي قابل للتشغيل.
* `chmod u+x <script.sh>` (أو `chmod 700 <script.sh>`): خطوة إجبارية قبل ما تشغل أي إسكربت مباشرة. الأمر ده بيدي صلاحية التنفيذ (Execute Permissions) للمستخدم الأساسي. من غيرها، الإسكربت هيفضل مجرد ملف قراءة ومش هينفع يشتغل بشكل مباشر.

### طرق تنفيذ الإسكربتات (Execution Methods)
الطريقة اللي بتشغل بيها الإسكربت بتأثر جداً / قوي على ازاي بيئة العمل معاك بتتعامل مع الـ Variables:

1. **Absolute Path/النسبي (`/home/user/script.sh` أو `./script.sh`):**
    * بتحتاج إن الملف يكون معمول له صلاحية تشغيل (+x). 
    * الفكرة هنا إن التيرمينال بتفتح "Shell ابنة" أو (Child Shell) جديدة خالص وبتشغل الأوامر دي جواها. وأول ما الإسكربت يخلص.. الـ Shell دي بتتمسح بكل متغيراتها المؤقتة (Local variables) وترجع للتيرمينال الأصلية كأن شيئاً لم يكن.
2. **استدعاء المترجم مباشرة (`bash script.sh` أو `sh script.sh`):**
    * مش محتاج صلاحية تشغيل (+x).
    * نفس الكلام، بيفتح Shell ابنة جديدة (Child Shell) وينفذ الشغل فيها.
3. **التضمين أو الاستدعاء (`source script.sh` أو `. script.sh`):**
    * مش محتاج صلاحية تشغيل (+x).
    * هنا إنت **مبتفتحش** Shell ابنة. بل التيرمينال نفسها، في الجلسة اللي إنت واقف فيها حالياً، هي اللي بتقوم بتلقين سطور الإسكربت وتنفيذها مباشرة (Current shell environment). فايدة ده إن لو الإسكربت بيعرّف Variables أو Functions، هتفضل موجودة وشغالة معاك بعد ما الإسكربت يخلص.

### تظبيط Path البحث المسبق (`$PATH`)
لو عايز تكتب اسم الإسكربت بتاعك في التيرمينال `my_script` وهو يشتغل مباشرة من غير ما تضطر تكتب المسار بتاعه `./my_script`، لازم تضيف مكان الفولدر اللي جواه الإسكربت لمتغير المسارات العالمي `$PATH`:
```bash
export PATH=$PATH:/path/to/your/script_directory
```
* عشان التعديل ده يفضل ثابت معاك دايماً ومتضطرش تكتبه كل ما تفتح تيرمينال جديدة أو تعمل ريستارت၊ ممكن تحط السطر ده آخر ملف الإعدادات الخاص بالـ Shell بتاعتك، زي ملف `~/.bashrc` أو `~/.bash_profile`.
