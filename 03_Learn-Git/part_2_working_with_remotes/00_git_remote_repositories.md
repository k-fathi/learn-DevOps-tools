# Remote Repositories

## 1. why remote repositories?
- Local-only work: Exists only on your machine
- No collaboration: Teammate can't access your code 
- Independent changes: Leads to fonflicts during merge
- No backup: Risk of losing work


## 2. what is remote 
A remote is a reference to a repository hosted on a server (e.g., GitHub, GitLab, Bitbucket). It allows you to:

- **Push** your local changes to share with others
- **Pull** changes made by teammates
- **Sync** code across multiple machines
- **Backup** your work in a central location

## 3. Types of remote repositories
1. Private repos
    - **Access**: Restricted to specific users or teams and collaborators only.
    - **Visibility**: Not visible to the public.
    - **Write access**: Only granted to authorized users and approved contributors.
    - **Usage**: Ideal for sensitive projects, proprietary code.
    - **Examples**: Backend service, infrastructure configs, internal tools.
2. Public repos
    - **Access**: Accessible to anyone with the URL.
    - **Visibility**: Visible to the public.
    - **Write access**: Usually open to contributions from the community.
    - **Usage**: Ideal for open-source projects, documentation, and educational resources.
    - **Examples**: K8s, Terraform, Public course repos.
