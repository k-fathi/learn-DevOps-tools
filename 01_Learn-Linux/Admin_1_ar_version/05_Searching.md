# 05: البحث في لينكس (Searching)

## 1. مقدمة
لينكس فيه أدوات بحث قوية جداً تخليك تلاقي أي ملف، أو أمر، أو حتى كلمة جوه ملف، في ثواني.

## 2. البحث عن الأوامر
الأدوات دي بتعرفك مكان أوامر السيستم فين.

### `whereis`
بيجيبلك مكان الملف التنفيذي (Binary)، والسورس، وملف المانيوال.
```bash
whereis ls
# Output: ls: /usr/bin/ls /usr/share/man/man1/ls.1.gz
```

### `whatis`
بيجيبلك سطر واحد بيوصف الأمر ده بيعمل إيه.
```bash
whatis ls
# Output: ls (1) - list directory contents
```

## 3. البحث عن الملفات

### `locate` (السريع)
- **فكرته:** بيبحث في قاعدة بيانات جاهزة (`mlocate.db`).
- **ميزته:** صاروخ 🚀.
- **عيبه:** لو لسه عامل ملف حالا، مش هيلاقيه غير لما الداتابيز تتحدث.
- **تحديث الداتابيز:** `sudo updatedb`.

```bash
locate *.conf
```

### `find` (الدقيق)
- **فكرته:** بيبحث في الهارد ديسك لايف (Real-time).
- **ميزته:** دقيق جداً وفيه فلترز كتير (بالحجم، بالوقت، باليوزر..).
- **عيبه:** أبطأ شوية من `locate`.

**الطريقة:**
```bash
find <المكان> <المواصفات> <الأكشن>
```

**أشهر المواصفات (Criteria):**
- `-name "pattern"`: بالاسم (حساس للحروف).
- `-iname "pattern"`: بالاسم (مش فارقة كابيتال ولا سمول).
- `-type f`: ملفات بس.
- `-type d`: فولدرات بس.
- `-size +10M`: أكبر من 10 ميجا.
- `-user <name>`: تبع يوزر معين.
- `-mtime -7`: اتعدل في آخر 7 أيام.

**مثال:**
```bash
# هات كل ملفات اللوج (.log) في /var/log اللي اتعدلت النهاردة
find /var/log -name "*.log" -mtime -1
```

## 4. البحث جوه الملفات (`grep`)
الـ `grep` ده "الباشا" بتاع البحث في النصوص.

**الطريقة:**
```bash
grep [options] "pattern" [file]
```

**أهم الاختيارات:**
- `-i`: مش فارقة كابيتال ولا سمول (Ignore case).
- `-r`: بحث في كل الفولدرات والملفات (Recursive).
- `-v`: هات اللي **مش** بيطابق (اعكس البحث).
- `-n`: وريني رقم السطر.
- `-l`: اعرض اسم الملف بس، مش المحتوى.

**مثال:**
```bash
# دور على كلمة "error" في كل ملفات اللوجات
grep -r -i "error" /var/log
```

## 5. معرفة نوع الملف
في لينكس، الامتداد (Extension) مش مهم زي الويندوز. استخدم `file` عشان تعرف نوع الملف الحقيقي.

```bash
file my_script.sh
# Output: Bourne-Again shell script, ASCII text executable
```

## 4. 🏆 مثال من سوق العمل: تنظيف السيستم أوتوماتيك
هنستخدم `find` مع `xargs` عشان نعمل عملية تنظيف ذكية.

**السيناريو:** عايز تلاقي كل الملفات المؤقتة (`.tmp`) اللي في فولدر التطبيق `/var/app`، بشرط تكون **أكبر من 50 ميجا** ومحدش لمسها بقاله **أكتر من 30 يوم**، وتمسحهم.

```bash
find /var/app -name "*.tmp" -type f -size +50M -mtime +30 -print0 | xargs -0 rm -v
```

### شرح الأمر:
1.  **`/var/app`**: هدور فين.
2.  **`-name "*.tmp"`**: اسمها بينتهي بـ .tmp.
3.  **`-type f`**: ملفات بس (مش فولدرات).
4.  **`-size +50M`**: أكبر من 50 ميجا.
5.  **`-mtime +30`**: بقالها شهر محدش عدلها.
6.  **`-print0`**: افصل النتيجة بـ `null` (عشان لو الاسم فيه مسافات ميعملش مشكلة).
7.  **`| xargs -0`**: خد النتيجة دي وسلمها للأمر اللي جاي.
8.  **`rm -v`**: امسح واكتبلي مسحت إيه.

> **ليه `xargs`؟** عشان أسرع وأخف على السيستم من `-exec`، لأنه بيجمع الملفات ويبعتها للأمر `rm` مرة واحدة، مش مرة لكل ملف.
