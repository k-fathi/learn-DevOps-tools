# 29: الـ Wildcards والـ Regex

## 1. مقدمة
في لينكس، البحث ومطابقة النصوص (Pattern Matching) فن. لازم تفرق بين حاجتين بيلخبطوا ناس كتير:
1.  **Wildcards (Globbing):** دي بتاعة الـ **Shell**، وبنستخدمها مع **أسماء الملفات** (زي لما تقول `ls *.txt`).
2.  **Regular Expressions (Regex):** دي بتاعة **أدوات النصوص** (grep, sed)، وبنستخدمها عشان ندور جوه **محتوى الملفات** (زي كلمات معينة).

## 2. الـ Wildcards (مخصصة للملفات)
الشيل بيفك الرموز دي **قبل** ما ينفذ الأمر.

| الرمز | المعنى | مثال | يطابق إيه |
| :--- | :--- | :--- | :--- |
| `*` | أي عدد من الحروف | `*.txt` | كل ملفات الـ txt |
| `?` | حرف واحد بس | `file?.txt` | `file1.txt`, `fileA.txt` |
| `[]` | مجموعة حروف | `file[1-3].txt` | `file1` أو `file2` أو `file3` |
| `{}` | قائمة | `file{A,B}.txt` | `fileA.txt` و `fileB.txt` |

**أمثلة:**
```bash
rm temp[123].txt        # امسح temp1 و temp2 و temp3
cp file{1,2}.txt back/  # انسخ file1 و file2
```

## 3. الـ Regex (مخصصة للنصوص)
دي لغة البحث جوه الكلام. أقوى بكتير.

| الرمز | المعنى | مثال | يطابق إيه |
| :--- | :--- | :--- | :--- |
| `.` | أي حرف واحد | `a.c` | `abc`, `adc`, `a9c` |
| `^` | أول السطر | `^Error` | السطور اللي بتبدأ بـ Error |
| `$` | آخر السطر | `done$` | السطور اللي بتنتهي بـ done |
| `*` | تكرار اللي قبله (0 أو أكثر) | `ab*` | `a`, `ab`, `abb`, `abbb` |
| `+` | تكرار اللي قبله (1 أو أكثر) | `ab+` | `ab` و `abb` (بس مش `a`) |

**أمثلة:**
```bash
grep "^Error" syslog     # هات السطور اللي بتبدأ بكلمة Error
grep "fail$" log.txt     # هات السطور اللي بتخلص بكلمة fail
grep "port [0-9]\+" conf # هات كلمة port اللي وراها أرقام
```

## 4. الفروقات الجوهرية
| الميزة | Wildcards | Regex |
| :--- | :--- | :--- |
| **المستخدم** | الـ Shell (`ls`, `cp`, `rm`) | الأدوات (`grep`, `sed`, `vim`) |
| **الهدف** | أسماء الملفات | محتوى النصوص |
| **معنى `*`** | أي كلام في الدنيا | تكرار الحرف اللي قبله |

> [!IMPORTANT]
> **غلطة مشهورة:** متستخدمش Wildcards مع grep والعكس.
> - `grep *.log` ❌ (غلط، الشيل هيفكها لأسماء ملفات قبل ما grep يشوفها)
> - `grep "Error" *.log` ✅ (صح، دور على Error في كل ملفات اللوج)

## 5. 🏆 مثال من سوق العمل: تنظيم الملفات بذكاء
**السيناريو:** عندك فولدر `downloads` مليان ملفات، وعايز ترتبهم في فولدرات حسب السنة والنوع بأسلوب المحترفين.

```bash
# 1. انقل الصور اللي في اسمها 2023
mkdir -p 2023/images
# *2023*: أي حاجة قبلها 2023 وبعدها أي حاجة
mv *2023*.jpg 2023/images/

# 2. انقل ملفات الداتا data_a.csv لغاية data_e.csv
mkdir -p data/sets
# [a-e]: أي حرف من a لـ e
mv data_[a-e].csv data/sets/

# 3. اعمل فولدرات باك أب للـ 5 سنين الجايين
# {..}: اعمل تسلسل
mkdir backup_{2025..2029}
```

> **النتيجة:** بـ 3 أوامر رتبت مئات الملفات وعملت الهيكل التنظيمي بتاعك.
