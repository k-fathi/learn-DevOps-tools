# 10: المتغيرات في لينكس (Variables)

## 1. مقدمة
المتغيرات (Variables) هي "صناديق" بتشيل فيها داتا (كلام أو أرقام) عشان تستخدمها تاني. بتنقسم لنوعين: **متغيرات محلية** (خاصة بيك دلوقتي) و **متغيرات بيئة** (خاصة بالسيستم كله).

## 2. المتغيرات المحلية (User-Defined Variables)
دي متغيرات أنت بتعملها وبتعيش جوه الجلسة (Session) بتاعتك بس.
- **الطريقة:** `VAR_NAME=value` (مفيش مسافات حول الـ `=`).
- **الاستخدام:** `$VAR_NAME`.

```bash
name="Karim"
echo "Hello, $name"
# Output: Hello, Karim
```

## 3. متغيرات البيئة (Environment Variables)
دي متغيرات عامة بتأثر على سلوك السيستم والبرامج.
- **PATH:** الأماكن اللي الشيل بيدور فيها على الأوامر.
- **USER:** اليوزر الحالي.
- **HOME:** مسار الـ Home Directory بتاعك.
- **SHELL:** مسار الشيل اللي أنت شغال عليه.

```bash
echo $HOME
# Output: /home/karim
```

### متغير الـ `PS1` (شكل الـ Prompt)
ده اللي بيحدد شكل السطر اللي بتكتب فيه الأوامر.
```bash
echo $PS1
# Example output: \u@\h:\w\$
```
- `\u`: اسم اليوزر.
- `\h`: اسم الجهاز (Hostname).
- `\w`: المكان اللي أنت واقف فيه.

## 4. تصدير المتغيرات (Exporting)
في العادي، المتغيرات بتبقى **محلية**. عشان تخلي البرامج التانية أو السكربتات اللي هتشغلها تشوف المتغير ده، لازم تعمله `export`.

```bash
# متغير محلي (السكربتات مش هتشوفه)
my_var="local"

# متغير متصدر (السكربتات هتشوفه)
export MY_GLOBAL="global"
```

> [!NOTE]
> الـ `export` بيورث المتغير للأبناء (السكربتات)، بس الأبناء ميقدروش يعدلوا متغيرات الأب (الشيل الأصلي).

## 5. إدارة المتغيرات
| الأمر | الوصف |
| :--- | :--- |
| `set` | اعرض كل المتغيرات (محلية وعامة). |
| `env` / `printenv` | اعرض متغيرات البيئة بس (Exported). |
| `unset <name>` | امسح متغير. |

## 6. الزتونة (Summary)
- **Local Vars:** `VAR=value` (للشيل الحالي بس).
- **Env Vars:** `export VAR=value` (للشيل وأي حاجة يشغلها).
- **الاستمرارية:** عشان المتغير يفضل موجود علطول، ضيفه في `~/.bashrc` أو `/etc/environment`.

## 7. 🏆 مثال من سوق العمل: إعدادات التطبيقات (12-Factor App)
**السيناريو:** التطبيقات الحديثة (زي اللي في Docker) بتستخدم Environment Variables عشان الإعدادات، بدل ما تكتب باسووردات جوه الكود. هنجهز نص الاتصال بقاعدة البيانات.

```bash
# 1. حدد المتغيرات بشكل آمن (المسافة قبل الأمر بتمنعه يتسجل في الهيستوري)
 export DB_HOST="db.prod.local"
 export DB_USER="admin"
 export DB_PASS="s3cr3t!"

# 2. جمع نص الاتصال من المتغيرات دي
APP_CONN_STR="mysql://$DB_USER:$DB_PASS@$DB_HOST:3306/webapp"

# 3. اتأكد
echo "Connecting to: $APP_CONN_STR"
# Output: Connecting to: mysql://admin:s3cr3t!@db.prod.local:3306/webapp

# 4. خليه دائم لليوزر 'webapp' بس
echo 'export DB_HOST="db.prod.local"' >> /home/webapp/.bashrc
```

> **ليه؟** عشان لما تنقل من بيئة التطوير (Dev) للإنتاج (Prod)، كل اللي هتعمله إنك تغير قيمة المتغير من غير ما تلمس كود البرنامج.
