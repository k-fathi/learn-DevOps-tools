# 07: الأنابيب (Pipelining)

## 1. مقدمة
الـ **Pipe** (`|`) هي أقوى ميزة في اللينكس. بتخليك تربط الأوامر ببعض زي "خط الإنتاج". بتاخد اللي طالع من أمر (Output)، وتدخله كإنه (Input) للأمر اللي بعده.

## 2. الطريقة (Syntax)
```bash
command1 | command2 | command3
```
*الداتا بتمشي من الشمال لليمين.*

### شكل التوصيل
> ![Linux Pipes Workflow](screens/infographic_pipes.png)

## 3. أمثلة مشهورة
```bash
# فلترة النتائج: هات الملفات ونقي منهم اللي فيهم كلمة "conf"
ls -la | grep "conf"

# العد: عد عدد الملفات في الفولدر
ls | wc -l

# الترتيب: اعرض العمليات الشغالة ورتبهم حسب استهلاك الذاكرة
ps aux | sort -nk 4
```

## 4. الأمر `xargs` (الوصلة)
في أوامر (زي `rm`, `kill`, `mkdir`) مبتفهمش الـ Input اللي جاي من الـ Pipe (مبتقراش تيار نصوص)، هي مستنية "Arguments" صريحة. هنا بييجي دور `xargs` عشان يحول الـ Input لـ Arguments.

### مقارنة بين الـ Pipe العادية و `xargs`
| الحالة | الأمر | النتيجة |
| :--- | :--- | :--- |
| **بيققبل stdin** | `cat file | grep "text"` | شغال (grep بتفهم تيارات النصوص) |
| **مبيقبلش stdin** | `echo file | rm` | مش هيشتغل (rm مش فاهمة تمسح إيه) |
| **باستخدام xargs** | `echo file | xargs rm` | شغال (xargs خدت كلمة "file" وادتها لـ rm كأنك كتبتها بايدك) |

### استخدامات `xargs`
```bash
# امسح كل ملفات .log اللي الـ find لقاها
find . -name "*.log" | xargs rm

# اعمل فولدرات بأسماء مكتوبة جوه ملف تكست
cat folders.txt | xargs mkdir
```

## 5. الزتونة (Key Takeaways)
- **الـ Pipe (`|`)** بتوصل المخرج بالمدخل.
- **التوجيه (`>`)** بيوصل المخرج بملف.
- استخدم **`xargs`** لو الأمر التاني "غبي" ومبيفهمش الـ Pipe المباشر.
