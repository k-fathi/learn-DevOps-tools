# 19: نقل الملفات (Transferring Files)

## 1. مقدمة
نقل الملفات بين السيرفرات بشكل آمن دي حاجة يومية. بنستخدم أدوات بتعتمد على SSH زي `scp` و `rsync`.

## 2. SCP (Secure Copy)
أداة بسيطة لنقل الملفات (شبه `cp` بس عبر الشبكة).
*ملحوظة: مبقتش مفضلة أوي دلوقتي، والناس بتستخدم rsync بدالها.*

```bash
# ارفع ملف من جهازك للسيرفر (Upload / Push)
scp file.txt user@remote:/tmp/

# نزل ملف من السيرفر لجهازك (Download / Pull)
scp user@remote:/var/log/syslog .

# انقل فولدر كامل
scp -r folder/ user@remote:/home/user/
```

## 3. SFTP (Secure FTP)
دي طريقة تفاعلية، بتفتح جلسة وتفضل تنقل فيها ملفات.
```bash
sftp user@remote
```
**أوامر جوه الجلسة:**
- `put file`: ارفع ملف.
- `get file`: نزل ملف.
- `ls`: اعرض ملفات السيرفر.
- `lls`: اعرض ملفات جهازي أنا (Local ls).

## 4. Rsync (الجوكر)
دي أقوى أداة لنقل الملفات. ذكية جداً لأنها بتنقل **التغييرات بس** (Deltas)، مش الملف كله كل مرة.

**الطريقة:**
```bash
rsync [options] source destination
```

**أهم الاختيارات (`-avzP`):**
- `-a`: وضع الأرشيف (بيحافظ على الصلاحيات والتواريخ والروابط والملكية). **مهم جداً**.
- `-v`: وريني التفاصيل.
- `-z`: اضغط الداتا وهي بتتنقل (سرعة أعلى).
- `-P`: اعرض شريط التقدم (Progress Bar).

**أمثلة:**
```bash
# زامن فولدر محلي مع السيرفر (Push)
rsync -avzP /local/dir/ user@remote:/remote/dir/

# هات باك أب من السيرفر (Pull)
rsync -avzP user@remote:/var/www/html/ /backup/html/
```

---

## 5. 🏆 مثال من سوق العمل: باك أب ذكي وموفر
**السيناريو:** عندك مشروع حجمه كبير، وعايز تاخد منه نسخة على سيرفر تاني `backup-server`. عايز تنقل بس الملفات اللي اتعدلت عشان توفر وقت وباقة، ولو مسحت ملف من عندك يتمسح من هناك (Mirror).

```bash
# --delete: امسح الملفات من هناك لو مكنتش موجودة هنا (مراية تبيكال)
rsync -avz --delete myproject/ user@backup-server:/var/backups/myproject/
```

> **تحذير:** خلي بالك من `--delete` دي. لو مسحت ملف بالغلط من عندك وشغلت الأمر، هيتمسح من الباك أب كمان للأبد!

## 6. الزتونة (Key Takeaways)
- ملف واحد سريع؟ استخدم **`scp`**.
- باك أب أو فولدرات كبيرة؟ استخدم **`rsync`**.
- عايز تتصفح وتنقل براحتك؟ استخدم **`sftp`**.
