# 25: تحليل وتخزين اللوجات (Logs)

## 1. مقدمة
اللوجات (Logs) هي "الصندوق الأسود" بتاع السيرفر. أي حاجة بتحصل، بتتكتب هناك. لو السيرفر وقع، أو حد حاول يخترقه، اللوجات هي أول مكان تبص فيه.

عندنا نظامين للوجات:
1.  **Rsyslog:** الطريقة القديمة (ملفات تكست عادية).
2.  **Systemd-Journald:** الطريقة الحديثة (ملفات Binary متطورة).

## 2. Rsyslog (النظام الكلاسيكي)
بيخزن اللوجات في ملفات نصية تقدر تقراها بـ `cat` أو `less`.
- **مكان الإعدادات:** `/etc/rsyslog.conf`
- **أهم الملفات في `/var/log/`:**
    - `syslog` أو `messages`: كوكتيل لكل حاجة بتحصل في السيستم.
    - `auth.log` أو `secure`: ده أخطر واحد، فيه مين دخل ومين كتب باسورد غلط (SSH, sudo).
    - `dmesg`: رسائل الكرنل (تعريفات الهاردوير أول ما الجهاز يفتح).
    - `boot.log`: لجات بداية التشغيل.

## 3. Systemd-Journald (النظام الحديث)
ده بيلم اللوجات بشكل منظم أكتر، بس مينفعش تقراه بـ `cat`. لازم تستخدم أداة `journalctl`.

### استخدام `journalctl`
```bash
# اعرض كل اللوجات (زي less)
journalctl

# تابع اللوجات لايف (زي tail -f)
journalctl -f

# هات الأخطاء والمصايب بس (Errors & Critical)
journalctl -p err

# هات لوجات خدمة معينة (زي SSH)
journalctl -u sshd

# فلتر بالوقت
journalctl --since "1 hour ago"
journalctl --since "today"

# أهم أمر في حياتك (لما خدمة تفشل)
journalctl -xe
# -x: بيشرحلك إيه اللي حصل واقتراحات للحل.
# -e: بينط لآخر اللوجات (عشان تشوف المشكلة اللي لسه حاصلة).
```

> [!TIP]
> **نصيحة ذهبية:** أول ما `systemctl` يقولك إن الخدمة فشلت (Failed)، اجري فوراً اكتب `journalctl -xe`. ده مفتاح الحل.

## 4. 🏆 مثال من سوق العمل: كشف محاولة اختراق
**السيناريو:** جالك إنذار إن في حد بيحاول يعمل Brute Force على السيرفر. عايز تطلع كل الـ IPs اللي حاولت تدخل غلط في **آخر ساعة** وتعد كل واحد حاول كام مرة.

```bash
journalctl -u ssh --since "1 hour ago" | grep "Failed password" | awk '{print $(NF-3)}' | sort | uniq -c
```

### شرح "التعويذة" دي:
1.  **`journalctl -u ssh`**: هات لوجات الـ SSH بس.
2.  **`--since "1 hour ago"`**: هات بتوع آخر ساعة بس.
3.  **`grep "Failed password"`**: نقي السطور اللي فيها فشل دخول.
4.  **`awk '{print $(NF-3)}'`**: هات الخانة الرابعة من اليمين (اللي هي الـ IP في اللوج).
5.  **`sort | uniq -c`**: رتبهم وعد كل IP اتكرر كام مرة.

> **النتيجة:** قائمة بأسماء "الحرامية" وعدد محاولاتهم، جاهزة عشان تديهم بلوك.

## 5. الحفاظ على اللوجات (Persistence)
افتراضياً، `journald` ممكن يمسح اللوجات لما تعمل ريستارت. عشان تحتفظ بيها:
1.  افتح ملف `/etc/systemd/journald.conf`.
2.  خلي `Storage=persistent`.
3.  اعمل ريستارت للخدمة: `systemctl restart systemd-journald`.

## 6. الزتونة (Key Takeaways)
- اللوجات القديمة = ملفات تكست في `/var/log`.
- اللوجات الحديثة = `journalctl`.
- مشاكل الدخول = `/var/log/secure` أو `journalctl -u ssh`.
- مشاكل الخدمات = `journalctl -xe`.
